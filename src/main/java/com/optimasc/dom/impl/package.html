Introduction

This is a very simple subset of the DOM Level 3 Core specification. Only Document, DocumentType, Element and Attr nodes can easily be manipulated.

Compliance

All methods required by JSR 287 are implemented and can freely be used, most methods of JSR 280 are implemented (non namespace aware methods of that JSR are not implemented, as they can be emulated by setting the namespaceURI to null). All the Namespace aware versions of the different public API's should be used, others are not implemented.

If a method is not implemented, it will raise a DOMException with an error code DOMException.NOT_SUPPORTED_ERR.

The following methods are partially implemented:

  • setUserData(String, Object, UserDataHandler) will raise an error if the UserDataHandler parameter is not null.
  • isEqualNode(Node) only does partial comparison for performance reasons, it does not compare the equality of all children nodes.

Details

For this implementation, the supported node types, and which node types they may have as children, are as follows:

  • Document -- Element (maximum of one)
  • Element -- Element, Text (maximum of one)
  • Attr -- no children
  • Text -- no children

To follow the specification, the Text node implementation interface is also exposed but only read-only as children of Element nodes. These Text nodes are internally created by calling setTextContent on the Element nodes, it is not possible to manually and append Text nodes.
